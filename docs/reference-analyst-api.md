# Analyst API and Data Model Reference

The Analyst panel is backed by four HTTP endpoints, three PostgreSQL tables, and two Python modules. This reference describes the public surface — types, defaults, error modes — exactly as implemented. For *why* the algorithms work the way they do, see [Explanation: analyst aggregation math](./explanation-analyst-aggregation-math.md).

## API endpoints

All endpoints under `/api/analyst/*` require the session token in the `x-session-token` header. The `/api/admin/analyst/ingest` endpoint requires `x-admin-token` (the `ADMIN_TOKEN` env var, defaults to `APP_PASSWORD`).

### `POST /api/admin/analyst/ingest`

Re-parses the `researcher/` directory and upserts all three tables. Idempotent.

**Request body**: none.

**Response 200**:

```json
{
  "status": "ok",
  "company_reports": {"parsed": 8, "errors": 0},
  "industry_chains": {"parsed": 3, "errors": 0},
  "company_list_rows": {"parsed": 113, "errors": 0}
}
```

**Errors**: 401 if `x-admin-token` missing or wrong; 500 if `researcher/` directory is missing.

### `GET /api/analyst/core-companies`

**Query parameters**:

| Name | Type | Required | Description |
|---|---|---|---|
| `as_of_date` | ISO date `YYYY-MM-DD` | yes | Business date for portfolio, snapshot, price data |

**Response 200**:

```json
{
  "as_of_date": "2026-05-29",
  "total_amount_cny": 5877305.92,
  "companies": [
    {
      "stock_code": "688041.SH",
      "stock_name": "海光信息",
      "report_available": true,
      "portfolio": {
        "weight_pct": 2.05,
        "amount_cny": 120511.32,
        "pe_ttm_dynamic": 38.6,
        "pb_mrq_dynamic": 5.6,
        "ps_ttm_dynamic": 12.3,
        "latest_close": 95.4,
        "latest_close_date": "2026-06-20"
      },
      "report_sections": {
        "market_focus": "...",
        "core_competence": "...",
        "supply_demand": "...",
        "marginal_change": "...",
        "valuation": "...",
        "risk": "..."
      }
    }
  ]
}
```

`portfolio` is `null` if the stock is not held on `as_of_date`. `weight_pct` divides by the **total** portfolio amount (including undrilled funds), not the drilled-only denominator. `latest_close` is `null` if neither the financial snapshot nor `PriceCache` has a price.

### `GET /api/analyst/stock/{code}`

**Query parameters**:

| Name | Type | Required | Description |
|---|---|---|---|
| `as_of_date` | ISO date | yes | Business date |
| `code` | path | yes | Stock code; URL-encode `.` as `%2E` or pass as-is (the route accepts the raw code) |

**Response 200**:

```json
{
  "stock_code": "688041.SH",
  "stock_name": "海光信息",
  "as_of_date": "2026-05-29",
  "portfolio_weight_pct": 2.05,
  "amount_cny": 120511.32,
  "pe_ttm_dynamic": 38.6,
  "pb_mrq_dynamic": 5.6,
  "ps_ttm_dynamic": 12.3,
  "latest_close": 95.4,
  "latest_close_date": "2026-06-20",
  "source_funds": [
    {
      "fund_code": "007818.OF",
      "fund_name": "国泰中证全指通信设备ETF联接C",
      "equivalent_shares": 1263.45,
      "fund_amount_cny": 120511.32,
      "ratio_in_portfolio_pct": 2.05,
      "ratio_in_fund_pct": 22.27
    }
  ],
  "report_sections": {"...": "..."}
}
```

`source_funds` is empty if the stock has no penetration rows. `equivalent_shares` is `null` if `latest_close` is missing. `ratio_in_fund_pct` is `null` if the fund's total holding amount is 0. Rows sorted by `fund_amount_cny` descending.

## Python modules

### `services/analyst_parser.py`

| Function | Returns | Used by |
|---|---|---|
| `parse_company_report(path: Path)` | dict with `success`, `stock_code`, `stock_name`, six `section_*` fields, `raw_text`, `source_file`, `parsed_at` | `parse_all` |
| `parse_chain_summary(path: Path)` | dict with `success`, `chain_name`, `narrative_md`, `source_file`, `parsed_at` | `parse_all` |
| `parse_chain_company_list(path: Path)` | dict with `success`, `chain_name`, `rows` (list of dicts matching `AnalystIndustryChainCompany`), `errors`, `source_file` | `parse_all` |
| `parse_all(researcher_dir: Path | None)` | `{company_reports, chain_summaries, chain_company_lists}` | `ingest_analyst_data` |

`parse_company_report` first tries `python-docx`, then falls back to direct XML extraction from `word/document.xml`. The fallback handles DOCX files generated by tools that omit standard `Heading` styles.

### `services/analyst_service.py`

| Function | Returns | Used by |
|---|---|---|
| `ingest_analyst_data(db, researcher_dir=None)` | dict with `status` and three section summaries | `POST /api/admin/analyst/ingest` |
| `get_core_companies(db, as_of_date)` | dict with `as_of_date`, `total_amount_cny`, `companies` | `GET /api/analyst/core-companies` |
| `get_stock_detail(db, code, as_of_date)` | dict with portfolio summary, `source_funds`, `report_sections` | `GET /api/analyst/stock/{code}` |
| `get_industry_chains(db, as_of_date)` | dict with `as_of_date`, `total_amount_cny`, `chains` | `GET /api/analyst/industry-chains` |

`get_industry_chains` calls `_compute_chain_portfolio_metrics` and `_compute_chain_csi300_metrics` internally. Both helpers return `None` for any valuation field where the input rows have missing or non-positive values.

## Frontend components

| Component | Role |
|---|---|
| `AnalystPanel.jsx` | Sub-tab bar (`核心公司` / `产业链`); fetches business date from `/api/data-version` |
| `CoreCompaniesPage.jsx` | Loads `/api/analyst/core-companies`, sorts by weight desc, renders `CompanyCard` grid |
| `CompanyCard.jsx` | Folded: code, name, weight, PE/PB/PS, latest close. Expanded: tabs for "具体持仓" (source-fund table) and "研究报告" (6-section pager). |
| `IndustryChainsPage.jsx` | Loads `/api/analyst/industry-chains`, renders `ChainCard` grid |
| `ChainCard.jsx` | Folded: 3-col comparison (asset vs CSI300). Expanded: tabs for "组合" (held-stock table) and "研究报告" (Markdown narrative). |
| `PriceBadge.jsx` | Inline badge: `close 95.40 (2026-06-20)` |
| `utils/markdown.js` | `mdToHtml`, `injectPriceBadges`, `highlightNumbers` (yellow span wrapper) |

## Frontend API client

`frontend/src/api.js`:

```js
getAnalystCoreCompanies(asOfDate)
getAnalystStockDetail(code, asOfDate)
getAnalystIndustryChains(asOfDate)
postAnalystIngest()
```

All return the JSON body directly. Errors propagate via Axios rejection.
