## 三、根因分析（云端同样的问题会怎么产生）

### 3.1 Bug #1：港股 shares_eq 用 HKD 当 CNY

**生产路径**：
```
drill_snapshot.py _get_fund_price():
    if as_of_date is not None:
        nav_row = db.query(FundDailyNav.nav).filter(
            FundDailyNav.fund_code == fund_code,
            FundDailyNav.trade_date == as_of_date,
        ).first()
        ...
drill_snapshot.py 价格生成循环:
    fx_rate = _get_fx_rate(db, currency, "CNY", as_of_date, ...)
    price_cny = float(price) * fx_rate if fx_rate else float(price)  ← 当 fx_rate=None 时不分币种
    shares_eq = fund_price * STOCK_RATIO * (weight_pct / 100.0) / price_cny  ← 分母用了 HK price
```

**触发条件**：`fund_price` 来自 `FundDailyNav.nav`，但当 `as_of_date < 业务日期`，fund_price 实际是「前一日」基金净值；
当 `fx_rate` 缺失（exchange_rates 表里没该日期数据），shares_eq 分母用 HK price 而非 HK × FX。

**云端如何产生**：
- 若 scheduler 启动时 `exchange_rates` 表还没数据，或某天 FX 抓取失败
- `drill_snapshot_for_date` 调用 → shares_eq 算错

### 3.2 Bug #2/3：FX 数据跳变

**生产路径**：
```
scheduler.py:1334 job_fetch_intraday_change_pct  # 5分钟一次
scheduler.py:1342 18:00 job_pull_fund_nav   # 拉基金净值
scheduler.py:1346 20:30 job_pull_fund_nav_evening
...  # 这些都没拉 FX
```

**FX 数据从哪里来**：没有专门的 FX 拉取 job，依赖：
- `crawlers/exchange_rates.py::update_rates_today` 在某些 crawl 路径中调用
- 或管理员手动 `INSERT`

**云端如何产生 0.92/7.18 跳变**：可能是某次 crawl API 返回了错误数据（比如某些数据源在周末/节假日的占位值），没有校验就写入了。

### 3.3 Bug #4：价格缓存缺失

**生产路径**：
```
job_backfill_gaps (20:00):
    for h in holdings:  ← 只遍历 Holding 表
        查缺哪几天 → 调 fetch_price_history 补
# 836 个 constituents 大多数不在 Holding 表中
# 所以 constituent 价格只靠初始 backfill_360days.py 一次性拉
# 一次性拉后没有 daily 增量
```

**云端如何产生**：
- 一次性 `backfill_360days.py` 跑完后
- 每日 scheduler 只补 holdings（基金本身）的价格
- 836 个 constituent 价格停在 backfill 当天

### 3.4 Bug #5/6：drill_snapshot stale 回退污染

**生产路径**：
```
generate_drill_snapshot_for_date(6/29, strict_mode=False):  ← 默认 strict_mode=False
    for code, price, stale in priced:
        if price is None:
            ... 跳过
        else:
            # 即使 stale=True 也照样写入（带 is_stale_price=True 标记）
            current_price = float(price)
            ...
            fund_rows.append(FundDrillSnapshot(
                ...
                current_price_cny=price_cny,  ← 这个是 stale 6/25 的价！
                is_stale_price=True,         ← 标记但前端可能忽略
            ))
```

**云端如何产生**：
- strict_mode=False 时（手动 trigger / 6/29 当时已过 2 次 cutoff 但 6/26/6/27 周末无价 → fallback 6/25）
- 所有 6/29 drill 行都标记 `is_stale_price=True`，但前端 `IndexDrillBaseTab` 没过滤

### 3.5 Bug #7：分析师页面口径不一致

**生产路径**：
```
analyst_service.get_core_companies():
    amount = SUM(FullHoldingSnapshot.amount_cny)  ← FullHoldingSnapshot 是另一份
    total = SUM(FullHoldingSnapshot.amount_cny)    ← 不是全持仓 emv
    metrics = AShareFinancialSnapshot(...).pe_ttm_dynamic  ← 不是 drill_snapshot
```

FullHoldingSnapshot 是按 user × as_of_date × stock_code 聚合的快照表，与「分析-全持仓」API（用 `Holding` 表聚合 + drill 数据）数据源不同。

**云端如何产生**：FullHoldingSnapshot 由 `rebuild_full_holding_snapshot_for_user` job 生成，可能与 full-holding-table API 不同步。

### 3.6 Bug #8/9：当日涨跌幅算法 + 旧日期回退

**生产路径**：
```
overview/intraday-change 服务（旧）:
    today_pc = date.today()
    snap_rows = FullHoldingSnapshot[user][today]
    for code, amt in snap_rows:
        if code in change_map:  ← change_map 来自 PriceCache.change_pct
            weighted_sum += (amt/total) * change_map[code]
# 缺 .OF 基金（NAV 在 FundDailyNav 不是 PriceCache）
# 缺 drilled stocks（PriceCache 里没有每个 drill row 的 change_pct）
```

**云端如何产生**：旧 endpoint 只覆盖 ~17 行（holdings 中的 direct_stock + 部分有 change_pct 的），drilled 和 .OF 都被排除。

**异常识别**：


**期望**：`stale = 0`（2 次拉取规则严守后不应有 stale）

```sql
-- 4.3.2 检查 current_price 与同日 PriceCache 一致性
SELECT d.stock_code, d.as_of_date,
       d.current_price_cny AS drill_cny,
       p.close_px AS pc_close,
       ROUND((d.current_price_cny - p.close_px) / p.close_px * 100, 2) AS diff_pct
```

```sql
-- 4.4.2 检查某日期的 shares_eq 是否用过正确 FX
WITH ctx AS (
    SELECT '00728.HK' AS stock_code, '021142.OF' AS fund_code,
           '2026-05-29'::date AS as_of
```

```python
def check_shares_eq_consistency(db, as_of_date: date, tolerance=0.001) -> list[str]:
    """检查 shares_eq 在不同基金 × 不同日期下是否一致。
    
    Returns: 异常描述列表（每项一条）
    """
    issues = []
    rows = db.execute(text("""
        SELECT fund_code, stock_code,
               COUNT(DISTINCT shares_equivalent) AS distinct_se,
               ROUND(MIN(shares_equivalent)::numeric, 8) AS min_se
        FROM fund_drill_snapshot
        WHERE as_of_date BETWEEN :start AND :end
        GROUP BY fund_code, stock_code
        HAVING COUNT(DISTINCT shares_equivalent) > 1
    """), {"start": as_of_date - timedelta(days=30), "end": as_of_date}).fetchall()
    for r in rows:
        issues.append(f"shares_eq 不一致: fund={r[0]} stock={r[1]} distinct={r[2]}")
    return issues


def check_fx_anomaly(db, lookback_days=10, threshold=0.05) -> list[str]:
    """检查 FX 是否有跳变（环比 >5%）。"""
    issues = []
    rows = db.execute(text("""
        SELECT from_currency, rate_date, rate,
               LAG(rate) OVER (PARTITION BY from_currency ORDER BY rate_date) AS prev_rate,
               ABS(rate - LAG(rate) OVER (PARTITION BY from_currency ORDER BY rate_date)) / 
                   NULLIF(LAG(rate) OVER (PARTITION BY from_currency ORDER BY rate_date), 0) AS chg
        FROM exchange_rates
        WHERE rate_date >= CURRENT_DATE - INTERVAL '30 days'
    """))
    for r in rows:
        if r[4] and r[4] > threshold:
            issues.append(f"FX 跳变: {r[0]} {r[1]} {r[3]}→{r[2]} ({r[4]*100:.1f}%)")
    return issues


def check_drill_stale_rate(db, as_of_date: date, threshold=0.1) -> list[str]:
    """检查 drill_snapshot 中 is_stale_price 占比。
    
    Returns: 异常描述
    """
    n_total = db.execute(text("""
        SELECT COUNT(*) FROM fund_drill_snapshot WHERE as_of_date = :d
    """), {"d": as_of_date}).scalar()
    n_stale = db.execute(text("""
        SELECT COUNT(*) FROM fund_drill_snapshot WHERE as_of_date = :d AND is_stale_price
    """), {"d": as_of_date}).scalar()
    if n_total > 0 and n_stale / n_total > threshold:
        return [f"stale 占比过高: {n_stale}/{n_total} ({n_stale/n_total*100:.1f}%)"]
    return []
```

```python
@track_run("data_integrity_check")
def job_data_integrity_check():
    """每日 22:00 跑，检查当日 drill_snapshot 的完整性。
    
    发现问题则写入 data_gap_report 表，让 admin 在面板上看到。
    """
    from services.data_integrity_checker import (
        check_shares_eq_consistency, check_fx_anomaly, check_drill_stale_rate
    )
    db = SessionLocal()
    try:
        today = date.today()
        issues = []
        issues += check_shares_eq_consistency(db, today)
        issues += check_fx_anomaly(db)
        issues += check_drill_stale_rate(db, today)
        
        if issues:
            for issue in issues:
                db.add(DataGapReport(
                    report_date=today,
                    issue_type="drill_integrity",
                    description=issue,
                    severity="high",
                ))
            db.commit()
            logger.warning("data_integrity_check: %d issues found", len(issues))
        else:
            logger.info("data_integrity_check: no issues")
    finally:
        db.close()
```
