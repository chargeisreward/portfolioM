# 远程主机连接方案

## 1. 主机信息

| 项目 | 值 |
|------|-----|
| 主机 IP | 43.130.62.66 |
| 用户名 | ubuntu |
| 认证方式 | SSH 密钥（ed25519） |
| 私钥路径 | `~/.ssh/id_ed25519` |
| 操作系统 | Ubuntu（云端） |
| Python 版本 | 3.10.12（无 venv，直接使用系统 Python） |

## 2. SSH 连接

### 2.1 基本连接命令

```powershell
ssh -i ~/.ssh/id_ed25519 -o StrictHostKeyChecking=no ubuntu@43.130.62.66
```

### 2.2 执行远程命令

```powershell
# 单条命令
ssh -i ~/.ssh/id_ed25519 -o StrictHostKeyChecking=no ubuntu@43.130.62.66 "ls /home/ubuntu/loopeng/"

# 多条命令（注意 PowerShell 不支持 &&，远程 shell 支持）
ssh -i ~/.ssh/id_ed25519 -o StrictHostKeyChecking=no ubuntu@43.130.62.66 "cd /home/ubuntu/loopeng && python3 -m pytest report_engine/tests/ --tb=short"
```

### 2.3 关键参数说明

| 参数 | 说明 |
|------|------|
| `-i ~/.ssh/id_ed25519` | 指定私钥文件路径 |
| `-o StrictHostKeyChecking=no` | 跳过主机密钥确认（首次连接不再提示） |

## 4. 文件同步方案

### 4.1 同步方式选择

| 方式 | 可用性 | 说明 |
|------|--------|------|
| rsync | ❌ Windows 不可用 | Windows 无原生 rsync |
| scp | ✅ 可用 | Windows OpenSSH 自带，推荐使用 |
| git | ❌ 云端无 git 仓库 | 云端 `/home/ubuntu/loopeng/` 不是 git 仓库 |

### 4.2 scp 上传命令

**上传单个文件：**

```powershell
scp -i ~/.ssh/id_ed25519 -o StrictHostKeyChecking=no "D:\trae_projects\loopengneering\report_engine\subscription_manager.py" ubuntu@43.130.62.66:/home/ubuntu/loopeng/report_engine/
```

**上传多个文件：**

```powershell
scp -i ~/.ssh/id_ed25519 -o StrictHostKeyChecking=no "D:\trae_projects\loopengneering\report_engine\subscription_manager.py" "D:\trae_projects\loopengneering\report_engine\delivery_logger.py" ubuntu@43.130.62.66:/home/ubuntu/loopeng/report_engine/
```

**上传整个目录（递归）：**

```powershell
scp -r -i ~/.ssh/id_ed25519 -o StrictHostKeyChecking=no "D:\trae_projects\loopengneering\report_engine\channels" ubuntu@43.130.62.66:/home/ubuntu/loopeng/report_engine/
```

### 4.3 scp 下载命令

```powershell
# 下载单个文件
scp -i ~/.ssh/id_ed25519 -o StrictHostKeyChecking=no ubuntu@43.130.62.66:/home/ubuntu/loopeng/report_engine/db.py "D:\trae_projects\loopengneering\report_engine\"

# 下载整个目录
scp -r -i ~/.ssh/id_ed25519 -o StrictHostKeyChecking=no ubuntu@43.130.62.66:/home/ubuntu/loopeng/report_engine/tests "D:\trae_projects\loopengneering\report_engine\"
```

### 4.4 路径注意事项

| 场景 | 格式 | 示例 |
|------|------|------|
| 本地路径（scp） | Windows 反斜杠 + 双引号 | `"D:\trae_projects\loopengneering\..."` |
| 远程路径 | Linux 正斜杠 | `/home/ubuntu/loopeng/...` |
| 本地路径（ssh 命令内） | 不涉及 | — |

> ⚠️ **重要**：scp 的本地路径必须使用 Windows 格式（`D:\...`），不能使用 Git Bash 格式（`/d/...`），否则会报 `No such file or directory` 错误。

## 5. 测试验证流程

### 5.1 标准验证流程（每次代码变更后）

```powershell
# 1. 本地运行测试
cd d:\trae_projects\loopengneering
python -m pytest report_engine/tests/ reasoning/tests/ --tb=short

# 2. 同步变更文件到云端（示例：同步 channels 目录）
scp -r -i ~/.ssh/id_ed25519 -o StrictHostKeyChecking=no "D:\trae_projects\loopengneering\report_engine\channels" ubuntu@43.130.62.66:/home/ubuntu/loopeng/report_engine/

# 3. 云端运行测试
ssh -i ~/.ssh/id_ed25519 -o StrictHostKeyChecking=no ubuntu@43.130.62.66 "cd /home/ubuntu/loopeng && python3 -m pytest report_engine/tests/ reasoning/tests/ --tb=short"
```

### 5.2 测试命令对照

| 操作 | 本地命令 | 云端命令 |
|------|----------|----------|
| 运行所有测试 | `python -m pytest report_engine/tests/ reasoning/tests/` | `python3 -m pytest report_engine/tests/ reasoning/tests/` |
| 运行单个测试文件 | `python -m pytest report_engine/tests/test_channels.py -v` | `python3 -m pytest report_engine/tests/test_channels.py -v` |
| Python 版本 | Python 3.14 | Python 3.10.12 |

### 5.3 已知差异

| 差异点 | 本地（Windows） | 云端（Ubuntu） | 影响 |
|--------|-----------------|----------------|------|
| Python 版本 | 3.14 | 3.10.12 | `asyncio.get_event_loop()` 在 3.14 已废弃，需使用 `asyncio.new_event_loop()` |
| aiohttp | 已安装 | 未安装 | `test_send_successResponse` 测试在云端跳过 |
| Shell | PowerShell 5 | bash | 本地用 `;` 分隔命令，远程用 `&&` |
| rsync | 不可用 | 可用 | 只能使用 scp 同步 |

## 7. 故障排查

### 7.1 SSH 连接失败

```powershell
# 检查私钥是否存在
Test-Path ~/.ssh/id_ed25519

# 检查网络连通性
Test-NetConnection 43.130.62.66 -Port 22
```

### 7.2 scp 路径错误

```
错误：scp : stat local "/d/trae_projects/...": No such file or directory
原因：使用了 Git Bash 路径格式（/d/...）
解决：改用 Windows 路径格式（D:\...）并加双引号
```

### 7.3 云端测试失败（ModuleNotFoundError）

```
错误：ModuleNotFoundError: No module named 'aiohttp'
原因：云端未安装该模块
解决：在测试中使用 pytest.importorskip("aiohttp") 跳过依赖该模块的测试
```

### 7.4 PowerShell 命令分隔符

```powershell
# ❌ 错误：PowerShell 不支持 &&
ssh ... "cd /path && python3 -m pytest"

# ✅ 正确：PowerShell 使用 ; 或远程 shell 内部使用 &&
ssh ... "cd /path; python3 -m pytest"
# 或
ssh ... 'cd /path && python3 -m pytest'  # 远程 bash 支持 &&
```
