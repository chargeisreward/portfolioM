# PortfolioM 项目规则

## 禁止项
- **禁止使用 mock 数据**（写死的 INITIAL 数组、假搜索结果、硬编码的价格/PE/市值等）
- **禁止使用种子数据**（mock initial values、seed fixtures、fake data arrays）
- **禁止前端硬编码任何业务数据**（证券代码、价格、PE、市值、涨跌幅、产业链穿透数等）

## 必须项
- 所有业务数据必须从后端 API 获取
- 新建模块/页面/列表时，必须先实现后端数据源，前端只做渲染
- 后端数据缺失时，前端显示"加载中"或"暂无数据"，**不得用占位符或默认值冒充真实数据**
- 搜索框、关注列表、KPI 数字等所有动态内容必须走后端 API
